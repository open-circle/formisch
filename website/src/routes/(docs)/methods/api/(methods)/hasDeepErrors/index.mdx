# hasDeepErrors

Checks whether a specific field or the entire form contains any errors by walking through the field store and all its descendants. The walk stops as soon as the first error is found, making this more efficient than collecting all errors. This is useful for error indicators in multi-tab or multi-section forms.

```ts
const hasErrors = hasDeepErrors<TSchema>(form);
const hasErrors = hasDeepErrors<TSchema, TFieldPath>(form, config);
```

## Generics

- `TSchema` <Property {...properties.TSchema} />
- `TFieldPath` <Property {...properties.TFieldPath} />

## Parameters

- `form` <Property {...properties.form} />
- `config` <Property {...properties.config} />

### Explanation

The `form` parameter is the form store to check for errors. The optional `config` parameter scopes the check to a specific field's subtree via a `path` (when omitted, the entire form is checked). The walk stops at the first error found.
